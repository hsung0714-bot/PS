# Bronze3 홀수

## 문제 요약
7개의 자연수가 주어졌을 때, 그 중 **홀수만 골라 합을 구하고**,  
그 중에서 **가장 작은 홀수**를 찾는 문제.  
만약 홀수가 하나도 없다면 -1을 출력한다.

## 접근 방법
처음에는 홀수만 골라내고, 그 값들을 따로 저장해 나열해야 하는지에 대해 고민했다.
이를 위해 배열을 사용할까 생각했지만, 단순히 홀수의 합과 최솟값만 구하면 되기 때문에 배열은 불필요하게 무겁다고 판단했다.

그래서 반복문 안에서 조건문을 활용해 해결하는 방법을 선택했다.
for문을 돌면서 입력값이 홀수일 경우에만 sum에 더하도록 했고, 동시에 최솟값도 갱신하도록 구성했다.

또한 모든 수가 짝수인 경우를 처리하기 위해, 홀수일 때만 sum에 더해진다는 점을 이용했다.
즉, 반복문이 끝난 뒤 sum이 0이라면 홀수가 존재하지 않는 것이므로 모든 수가 짝수임을 판단할 수 있었다.

## 핵심 포인트
- for 문 안에 조건문 2개를 사용해서 홀수의 합과 최솟값을 구해야함
- 홀수가 하나도 없는 경우 예외 처리 필요

## 실패 기록
- 최소값 초기화를 잘못해서 틀림

## 배운점
- 조건에 맞는 값만 처리하는 필터링의 중요성
- 최소/최대값 문제에서 초기값 설정 방법
